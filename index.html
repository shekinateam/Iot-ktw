기본 논리 연산

자동화를 만들 때 가장 먼저 알아야 할 개념이에요. 어떤 조건일 때 어떤 동작을 할지 정하는 규칙입니다.

AND
모든 조건이 참일 때만 실행
하나라도 빠지면 실행 안됨
🔒 자물쇠 두 개 — 둘 다 열어야 문이 열림
OR
조건 중 하나라도 참이면 실행
더 넓은 조건
🚌 버스 OR 지하철 중 하나 오면 → 출발
NOT
조건의 반대 상태일 때 실행
'아닐 때'로 이해하면 쉬움
NOT 감지됨 = 사람 없음

실생활 예시로 이해하기

연산자실생활 비유자동화 예시
AND배고프고 AND 돈 있어야 → 밥 먹음전등 ON AND 사람 없음 → 전등 끄기
AND비 오고 AND 우산 없어야 → 빌림온도 낮음 AND 오전 시간 → 난방 켜기
OR춥거나 OR 비 오면 → 외투 입기동작감지 OFF OR 재실 OFF → 타이머 시작
OR전화 OR 문자 오면 → 확인버튼1 OR 버튼2 누르면 → 조명 켜기
NOT우산 없으면(NOT 있음) → 챙기기NOT 동작 감지됨 → 사람 없음으로 판단
AND(4 input)4가지 조건 모두 맞아야 실행Timer경과+동작없음+재실없음+전등켜짐 → 끄기

NOT 연산 완전 이해하기

NOT은 처음엔 헷갈리는 분들이 많아요. 핵심은 딱 하나입니다. 입력값을 뒤집는다는 거예요.

NOT 의 동작 원리

입력 1 (ON / 참 / 감지됨) → 출력 0 (OFF / 거짓 / 감지 안됨)
입력 0 (OFF / 거짓 / 감지 안됨) → 출력 1 (ON / 참 / 감지됨)

왜 NOT이 필요한가요?

센서는 대부분 "감지됨 = 1" 을 출력해요. 그런데 자동화에서는 "사람 없을 때" 를 조건으로 쓰고 싶을 때가 많아요. 이럴 때 NOT으로 뒤집는 거예요.

센서 상태센서 출력값NOT 통과 후의미
사람 있음 (움직임 감지)1 (ON)0 (OFF)조건 불만족 → 실행 안됨
사람 없음 (움직임 없음)0 (OFF)1 (ON)조건 만족 → 실행됨

NOT 없이 vs NOT 있을 때 비교

❌ NOT 없이:
동작감지 → AND → 전등 끄기
→ 사람이 있을 때 전등이 꺼짐 (원하는 것과 반대!)

✅ NOT 있을 때:
동작감지 → NOT → AND → 전등 끄기
→ 사람이 없을 때 전등이 꺼짐 (정확!)

실전에서 NOT을 쓰는 대표 상황

상황센서 원래 출력NOT 적용 후자동화 조건
사람 없을 때 전등 끄기동작감지 = 0 (없음)NOT → 1사람 없음 조건 성립
문 닫혀있을 때 에어컨 켜기도어센서 = 0 (닫힘)NOT → 1문 닫힘 조건 성립
전등 꺼져 있을 때만 켜기전등 = 0 (꺼짐)NOT → 1꺼진 상태 조건 성립
에어컨 안 켜져 있을 때 알림에어컨 = 0 (꺼짐)NOT → 1미작동 상태 감지

NOT + AND 조합 — 가장 많이 쓰는 패턴

NOT은 단독으로 쓰기보다 AND와 함께 쓸 때 진짜 힘을 발휘해요.

예시: "전등이 켜져 있고, 사람이 없을 때 → 전등 끄기"

전등 상태(1=켜짐) ──────────────────────→ AND ─→ 전등 끄기
동작감지(0=없음) → NOT(0→1로 변환) ─→ AND
💡 핵심 요약
NOT = 0을 1로, 1을 0으로 뒤집기. "없을 때", "꺼져 있을 때", "안 감지될 때" 를 조건으로 쓰고 싶으면 NOT을 쓰세요!

언제 뭘 쓸까?

상황사용할 연산이유
여러 센서 중 하나라도 반응하면 시작OR넓게 트리거 잡기
최종 실행 전 여러 조건 확인AND오작동 방지
'없을 때', '꺼져 있을 때' 표현NOT반대 상태 표현
4가지 이상 조건 동시 확인AND(4 input)복잡한 자동화
💡 핵심 원칙
트리거는 OR로 넓게 잡고, 최종 실행 조건은 AND로 엄격하게 걸어야 오작동을 막을 수 있어요!

포맷 설정 상세

블록이 센서/기기 데이터를 어떤 형식으로 읽을지 설정합니다.

기본 포맷 4종

포맷의미권장 기기
mi단일 값 하나 (가장 기본)전등 스위치 ON/OFF
moid ★특정 속성 하나만 지정해서 읽기동작감지/재실/온도 센서
moids ★여러 속성 동시에 읽기스마트 플러그 (전력+전압+전류)
moid Array배열 형태로 여러 값 읽기다중 기기 값 묶음

실제 포맷 패턴 예시

기기포맷 패턴의미
동작감지 센서mi:io///^M%d*$M으로 시작하는 숫자값 (움직임)
재실 센서moid재실 감지 속성
온도 센서moidtemperature 속성
스마트 플러그moids:io///^O%d*$O로 시작하는 숫자값 (전력W)

정규식 패턴 해석법

moids : io /// ^O%d*$
  ↓       ↓             ↓
읽기방식  신호방식   필터 (정규식)

^ = 값의 시작
O = O로 시작하는
%d = 숫자 한 자리
* = 0개 이상 반복
$ = 값의 끝

→ 결국: O로 시작하고 숫자로 끝나는 값만 필터링
💡 예시
스마트 플러그는 전력(W)/전압(V)/전류(A) 여러 값을 동시에 보냄 → moids + ^O%d*$ 필터로 전력값만 쏙 뽑아 사용!

Reset 활용법 — 이게 핵심이에요!

사람이 중간에 다시 들어왔다 나갈 경우 10분을 처음부터 다시 카운트하려면 Reset을 연결해야 해요.

상황연결결과
동작감지 OFF 됨→ Timer Start카운트 시작
동작감지 ON 됨 (사람 다시 옴)→ Timer Reset카운트 0으로 초기화
재실 OFF 됨→ Timer Start카운트 시작
재실 ON 됨 (사람 감지)→ Timer Reset카운트 0으로 초기화
❌ Reset 없을 때:
동작감지 OFF → 10분 카운트 시작
5분 후 사람 다시 들어옴
5분 더 지나면 전등 꺼짐 → 오작동!

✅ Reset 있을 때:
동작감지 OFF → 10분 카운트 시작
5분 후 사람 다시 들어옴 → Reset → 0으로 초기화
다시 나가면 → 10분 처음부터 재카운트 → 정확!
⭐ 추천
Start와 Reset을 같이 쓰면 훨씬 정확한 자동화가 됩니다!

실전 패턴 1 — 두 센서로 전등 자동 끄기
🏠 이런 상황을 해결해요!
방에서 나갔는데 전등을 깜빡하고 안 껐을 때 → 10분 뒤 자동으로 꺼줌

왜 센서가 2개일까요?

센서 종류특징한계
동작감지 센서 (PIR)움직임을 감지가만히 앉아 있으면 감지 못함
재실 센서 (mmWave)미세한 움직임도 감지설치 위치 중요
2개 함께 사용서로 보완더 정확한 판단

전체 흐름

① 동작감지 OFF 또는 재실 OFF 발생
↓
② OR 블록 → 둘 중 하나라도 꺼지면 다음 단계로
↓
③ Timer 시작 → 10분(600초) 기다림
↓
④ AND(4 input) → 4가지 조건 동시 확인
↓
⑤ 조건 모두 맞으면 → 전등 OFF

자동화 완성 후 스크린샷을 여기에 추가하세요

Timer가 돌아가는 10분 동안 상황이 바뀔 수 있어요. 그래서 실행 직전에 다시 한번 확인해요.

전체 흐름

① Calendar → 매일 09:00 ~ 16:00 시간 조건 확인

설정온도 이상 → AND → 에어컨 켜기
설정온도 미만 → AND → 에어컨 끄기

Integer Compare 설정 방법

설정의미예시
A > B현재온도가 설정온도보다 높으면26도 이상이면 에어컨 켜기
A < B현재온도가 설정온도보다 낮으면24도 이하이면 에어컨 끄기
A = B정확히 같은 값잘 사용 안 함

↓
③ Timer 설정 시간 경과
↓
④ device control → 플러그 OFF
